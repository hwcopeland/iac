# `keda` namespace — KEDA event-driven autoscaling operator

[KEDA](https://keda.sh/) operator, installed **manually via Helm** on the live
cluster (no prior source in this repo). Codified here to make the install
reproducible from git. This directory does **not** convert the live release to a
Flux `HelmRelease` or RKE2 `HelmChart` CR — it documents the manual `helm`
install that actually produced the running operator, matching the repo
convention for manually-installed Helm components (cf. `rke2/monitor/`).

## Running release (source of truth, captured from cluster)

| Field | Value |
|---|---|
| Helm release | `keda` (namespace `keda`) |
| Chart | `keda` |
| Chart version | `2.19.0` |
| App version | `2.19.0` |
| Repo | `kedacore` → `https://kedacore.github.io/charts` |
| User-supplied values | **none** — `helm get values keda -n keda` → `null` (pure chart defaults) |

## Files

| File | Captures |
|---|---|
| `values.yaml` | Minimal values; pins operator/metrics/webhook image registries to chart defaults for 2.19.0. The live release used NO overrides, so an empty values file is equally valid. |

## Reproduce the running operator

```bash
helm repo add kedacore https://kedacore.github.io/charts
helm repo update

# Pinned to the running chart version. --create-namespace is safe on reinstall.
helm upgrade --install keda kedacore/keda \
  --namespace keda --create-namespace \
  --version 2.19.0 \
  -f values.yaml
```

To reproduce the live release *exactly* (which used no overrides), drop `-f values.yaml`:

```bash
helm upgrade --install keda kedacore/keda \
  --namespace keda --create-namespace \
  --version 2.19.0
```

## Notes / flags

- **No secrets** are referenced by this release; nothing to externalize via
  ExternalSecret here.
- KEDA's TLS certs for the metrics/webhook servers are **auto-generated** by the
  chart (`certificates.autoGenerated: true`, default) — not cert-manager-issued,
  so they do not touch the shared `cf-issuer` / gateway cert chain.
- The chart installs the KEDA CRDs (`crds.install: true`). On upgrade, review the
  KEDA release notes for CRD changes; Helm does not upgrade CRDs automatically in
  all chart versions.
- Version pin is `2.19.0`. Bump deliberately and review upstream changelog before
  changing it.
